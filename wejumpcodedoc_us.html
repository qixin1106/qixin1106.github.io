How to add custom training

Add custom training (operation flow)

  1. First you need a correct training configuration file (JSON)
  2. Copy the configuration file to the phone clipboard
  3. Reopen WeJump
  4. If the previous steps are correct, you will see a page where WeJump recognizes the new training action
  5. Click Save
  6. Go to training/training list/custom list to view

Code description

Code example

  • Custom training configuration files using JSON
  • You need to make sure that the configuration file is in the correct format.
{
  "descs" : "Your training project description here",
  "level" : 0,
  "name" : "Your training project name here",
  "items" : [
    {
      "bpm" : 0,
      "name" : "Ready",
      "type" : 2,
      "second" : 10
    },
    {
      "bpm" : 160,
      "name" : "Single under",
      "type" : 0,
      "second" : 30
    },
    {
      "bpm" : 0,
      "name" : "Rest",
      "type" : 1,
      "second" : 30
    },
    {
      "bpm" : 200,
      "name" : "Double under",
      "type" : 0,
      "second" : 30
    }
  ]
}

Parameter Description

  • name : String

The name of the training, limited to 30 characters, if it exceeds it will be truncated.

￼

  • descs : String

Description of the training project, limited to 140 characters, more than 140 characters will be truncated.

We recommend that you mention valuable information such as bpm, or training intensity in the description.

￼

  • level : Int

The training level is defined according to your preferences. Of course, you can define a very simple training as "difficult". We hope you define the level carefully. When you share the code with others, others can understand this training according to the level.

We have 5 training levels, If the level you set is not within this range, we will default to Easy.

0 is "Easy"
1 is "Medium"
2 is "Hard"
3 is "Hero"
4 is "Legend"

￼

  • items : Array

Here is the list of movements for your training plan.
The training action is limited to 30, if it exceeds, it will be truncated

{
      "bpm" : 0,
      "name" : "Ready",
      "type" : 2,
      "second" : 10
}
  • name : String

This is your action name, limited to 30 characters, if it exceeds it will be truncated.

￼

  • second : Int

The duration of each action (in seconds), the minimum value is 10s, the maximum value is 1800s (30 minutes), if not in this range we will set to the nearest valid value.

￼

  • bpm : Int

The bpm value is the number of jumps per minute. For example, if bpm is equal to 160, it means that 160 single-under jumps are required per minute. Of course, if it is Double-Under, you can set the bpm according to the number of jumps, or you can set the bpm according to the rotation of the rope.

The minimum value is 60 and the maximum value is 500. If the value you set is not in this range, the default will be set to the nearest valid value.

When type = 1 or type = 2, setting bpm is invalid. Write 0 by default, please refer to the code example.

  • type : Int

There are 3 types

0 is Action (This is the type marked as training)
1 is Rest
2 is Ready

The difference between Rest and Ready, we hope you have a preparation time when you start training, for example, if you click to start training, you need to put down your phone, and then pick up the skipping rope, we recommend your action list, the first one is Ready.

Different types will be marked with different colors.

Metronome won't start when type = 1 or tpye = 2.
